Django框架MTV流程回顾（上）

MVC模式：

代表的是Model（模型）、View（视图）、Controller（控制）。

Model：数据层，做程序的逻辑处理。

View：视图层， 用于向用户端展示数据。

Controller：控制层， 用于处理用户的交互，将用户界面与业务逻辑合理组织。

MTV模式：

Django框架的模型，分别代表了Model（模型）、View（视图）、Template（模板）

Model（模型层）：数据库相关操作（ORM）。

Template（模板层）：模板语法，充当数据展示角色。

View（视图层）：做程序的逻辑处理。

urls（路由分发）：按请求路径分发不同的views视图处理。

Web服务器理解：uWSGI与Nginx配置

Web开发中是由Web服务器与Web框架两大块构成。

其中Web服务器即建立连接，转发请求与响应，对请求与响应的处理交由Web框架处理。

WSGI： Web服务器网关接口（Python Web Server Gateway Interface）

是Python程序或框架和Web服务器之间的一种接口、协议

uWSGI：

实现了WSGI协议的一个Web服务器。

实现过程：用户到服务器间是http协议，服务器到Python框架间是WSGI协议，这里就需要uWSGI服务器实现WSGI协议。同理反之，响应数据也由Python框架通过WSGI协议返回给uWSGI服务器，再通过http协议传递给用户。

  • uWSGI + Django 即完成了简单的Web开发（仅适用访问量较少的个人网站）。

Nginx安装及配置

Nginx基础认识

正向代理：用户通过访问代理服务器，访问不能访问的网站。（例：VPN）

反向代理：用户只能访问代理服务器，不知道目标服务器的地址，降低服务器的负载。

区别：正向代理清楚服务器地址，反向代理只能访问暴露的Nginx服务器
