- 会话技术：

Http在Web中基本为短连接，请求的生命周期从request到response。

  1. Cookie 客户端会话技术

    不同域名、ip、浏览器都是隔离的；

    支持过期时间：默认浏览器关闭失效，max_age=过期时间（秒），None 不过期， expires 时间戳

  2. Session 服务端会话技术

    默认存储在服务器内存中，Django默认把session持久化数据库中，供多台服务器访问，

    session_key 主键；

    session_data 数据：使用Base64前部混淆串做数据安全；

    session_expires 过期时间 Django中默认14天。

    注销、删除（Session与Cookie一起）

  3. token 服务端会话技术（自定义session）

    Web页面与session基本一致

    移动端以JsonResponse传递，移动端自己存储

    • 用法：

      直接在用户表添加一字段，用来存储Token值；

      将Token存储在缓存中。

  • 三种会话技术对比
    1. Cookie 简洁，服务器压力小，存储在客户端相对不安全；
    2. Session 服务器需要维护，相对安全，Session依赖于Cookie;
    3. Token 拥有所有Session优点，支持更多终端，维护略麻烦。
