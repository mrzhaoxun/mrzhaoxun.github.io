Nginx安装及配置

Nginx基础认识

正向代理：用户通过访问代理服务器，访问不能访问的网站。（例：VPN）

反向代理：用户只能访问代理服务器，不知道目标服务器的地址，降低服务器的负载。

区别：正向代理清楚服务器地址，反向代理只能访问暴露的Nginx服务器

下载安装

  1. 编译安装，下载地址nginx.org

    install gcc openssl pcre zlib #四个依赖库

    安装：

    ./configure –prefix=/usr/local/nginx #指定安装路径

    编译（gcc）：

    make

    make install

  2. 或直接apt安装，无需编译安装

    apt install nginx

启动

#可通过配置文件启动（-t 测试）

nginx -t -c /user/loacl/nginx/conf/nginx.conf

#重启

nginx -s reload

#查看进程

ps -ef | grep nginx

#由master主进程：读取配置文件，维护子进程worker对请求作实际处理

关闭

优雅关闭(不会影响已经发起的请求)

kill -QUIT 主pid

快速关闭

kill -TERM 主pid

也可用

systemctl status/ start/ stop/ enabble/ disable nginx

nginx.conf 配置文件

负载均衡（基础）

仅一个域名入口，需将请求基本均匀的分发到不同服务器上。

​ 硬件负载均衡：贵

​ 软件负载均衡：Nginx、LVS、HAProxy；

通过配置nginx.conf实现：

  1. http模块中添加upstream模块，起名，设置后端服务器weight权重，检查后端服务器健康状况。

    ​ upstream www.web.com {

    ​ server 127.0.0.1:9100 weight=3;

    ​ server 127.0.0.1:9200 weight=1;

    ​ }

  2. server模块中添加转发到upstream模块名

    ​ location /web {

    ​ proxy_pass http://www.web.com

    ​ }
