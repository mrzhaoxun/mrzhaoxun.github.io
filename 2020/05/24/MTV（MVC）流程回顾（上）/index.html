Django框架MTV流程回顾（上）

MVC模式：

代表的是Model（模型）、View（视图）、Controller（控制）。

Model：数据层，做程序的逻辑处理。

View：视图层， 用于向用户端展示数据。

Controller：控制层， 用于处理用户的交互，将用户界面与业务逻辑合理组织。

MTV模式：

Django框架的模型，分别代表了Model（模型）、View（视图）、Template（模板）

Model（模型层）：数据库相关操作（ORM）。

Template（模板层）：模板语法，充当数据展示角色。

View（视图层）：做程序的逻辑处理。

urls（路由分发）：按请求路径分发不同的views视图处理。

  • Django流程：客户端 → urls → views → models → mysql
  • ​ 客户端 ← template ← views ← models←

Model 模型层：

ORM对象关系映射：将业务逻辑和SQL语句解耦合

  1. 定义数据表结构 DDL

    映射（迁移）前提：数据库配置完毕：setting设置，pymysql驱动安装，伪装

​ #mysql / base.py注释掉报错行

  • 迁移原理：

​ 1. 生成迁移文件：根据mdels与已经生成的迁移文件对比，生成新的迁移文件。

​ 2. 执行迁移： 查看迁移记录（表），执行未迁移过的文件。

​ 重新迁移：删除迁移文件、迁移完成的表、迁移记录。

  1. 增删改查 DML

    • 增、改： save()

      实例化，设置属性。→ save()

    • 删： delete()

      基于查询.delete()

    • 查： 模型类.objects.操作（条件）

      objects封装的操作：

      • Queryset：all()，filter()，exclude()，order_by()，values()，切片（不支持负数）
      • 获取单个对象：get() （多个/无 → 异常）， first()，last()
      • 内置函数：count()，exists()
      • 聚合函数：Sum()，Avg()，Count()，Max()，Min()

      条件：属性__操作符=值 （两 ‘_’）

      • gt，gte， it， ite， in， exact（=）
      • like(模糊查询)：contains，startswith，endswiths

      F（）对象 — 获取属性的值，用于不同属性间的运算，支持算术运算

      Q（）对象 — 对条件进行封装，支持逻辑运算 & | ~

最重要的表为主表。

on_delete=model.CASECASD（级联删除）

​ PROTECT（主表受从表保护）

​ SET（_NULL / _DEFAULT / ()）

一、1：1

​ 应用场景：复杂表拆分、拓展新功能。

​ models模块中的 OneToOnefield()

​ 原理：foreign key + unqiue

  • 级联数据获取：

​ 主获从：隐性属性（主表.从表小写）

​ 从获主：显性属性（从表.外键字段）

二、1：M

​ models模块中的 ForeignKeyField()

  • 级联数据获取：

    主获从：隐性属性（主表.从表_SET）

    从获主：显性属性（从表.外键字段）

三、M：M

​ models模块中的 MangToMangField()

​ 原理：产生一张单独的关系表（id， 主_id， 从_id）

​ 1. 关系表中存储关联表的主键，通过多个外键实现。

​ 2. 关系表中个多个外键值 组合唯一约束

  • 级联数据获取：

    主获从：隐性属性（主表.从表_SET）

    从获主：显性属性（从表.外键字段）

四、模型继承

​ 默认继承方法：通用字段在父表中，子表用外键连接父表。

​ 缺陷：关系型数据库越复杂，效率低；每个子数据都对应一个父数据，父表庞大。

  • 解决：元信息使父类模型抽象化 → 子表直接包含父类字段
